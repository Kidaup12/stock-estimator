<objective>
Wire Supabase Auth via `@supabase/ssr` end to end: the three cookie clients (browser/server/middleware), a root `middleware.ts` that refreshes the session cookie on every request AND gates unauthenticated traffic (401 on `/api/*`, redirect to `/login` on `/shop/*`), a custom branded `/login` page (magic link primary, Google OAuth secondary), the `auth/callback` code-exchange route, and a `signout` route.

Purpose: This is the authentication substrate (AUTH-01..05). It is code-independent of the schema/tenant work but creates `middleware.ts` — which Plan 03 will extend with tenant-header injection, so Plan 03 runs in a LATER wave (no shared-file race per Roy's standing rule).
Output: `lib/supabase/{client,server,middleware}.ts`, `middleware.ts`, `app/login/page.tsx`, `app/auth/callback/route.ts`, `app/auth/signout/route.ts`, three new npm deps.

This plan has a checkpoint: magic-link + Google OAuth + redirect-URL config is dashboard-only and must be human-verified.
</objective>

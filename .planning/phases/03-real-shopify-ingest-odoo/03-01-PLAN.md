  <behavior>
    - decrypt(encrypt("shpat_abc123")) === "shpat_abc123" (exact round-trip)
    - encrypt(x) !== x and the ciphertext string does not contain the plaintext substring
    - encrypt(x) called twice on the same input yields DIFFERENT ciphertext (random IV per call)
    - decrypt() on a tampered ciphertext (flip one char of the authTag/body) THROWS (GCM auth failure)
    - encrypt/decrypt read the key from process.env.TOKEN_ENCRYPTION_KEY (set a 32-byte base64 test key in the test via beforeAll)
  </behavior>
  <action>
    Create `lib/crypto/encryption.ts` using Node's built-in `crypto` (no new dependency). Implement authenticated AES-256-GCM per RESEARCH §Schema/D-05:
    - Derive the key once: `const key = Buffer.from(process.env.TOKEN_ENCRYPTION_KEY!, "base64");` and assert `key.length === 32` (throw a clear error otherwise — a misconfigured key must fail loud, not silently corrupt tokens).
    - `encrypt(plaintext: string): string` — generate a random 12-byte IV (`crypto.randomBytes(12)`), `crypto.createCipheriv("aes-256-gcm", key, iv)`, update+final to get ciphertext, read `cipher.getAuthTag()`. Return a single self-describing string: `[iv, authTag, ciphertext].map(b => b.toString("base64")).join(":")` (store iv+authTag+ciphertext together so decrypt is self-contained).
    - `decrypt(payload: string): string` — split on ":", base64-decode the three parts, `crypto.createDecipheriv("aes-256-gcm", key, iv)`, `decipher.setAuthTag(authTag)`, update+final. The final() call throws if the authTag does not verify (this gives the tamper-detection behavior).
    - Read the key lazily INSIDE the functions (not at module top-level) so the module imports cleanly even before env is set, and so the test's beforeAll-set key is picked up.
    Then create `lib/crypto/encryption.test.ts` implementing the five behaviors above. In `beforeAll`, set `process.env.TOKEN_ENCRYPTION_KEY = Buffer.from("0123456789abcdef0123456789abcdef").toString("base64")` (exactly 32 bytes).
  </action>

## Gotchas to remember

- **Stop the dev server before any `prisma migrate`/DB script** — Supabase free-tier session pooler has a tight connection cap; the running dev server starves CLI scripts (`Can't reach database server` even though TCP is open).
- Migrations: `prisma migrate diff --from-schema-datasource prisma/schema.prisma --to-schema-datamodel prisma/schema.prisma --script > migration.sql` then `migrate deploy`.
- `on_hand` NOT `available`.
- HMAC on `request.text()` BEFORE `request.json()`.
- Delete the mock `lib/shopify/client.ts` + remove its `eslint.config.mjs` allow-list entry when the real client lands.
- Nothing is pushed to the `Kidaup12/stock-estimator` upstream yet — all on the local `main`.

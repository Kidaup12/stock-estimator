    - verifyShopifyHmac(rawBody, "wrong/tampered-digest") === false
    - verifyShopifyHmac(tamperedBody, digestForOriginalBody) === false (one byte changed in the body invalidates it)
    - A header of different length than the digest returns false WITHOUT throwing (the length check guards timingSafeEqual, which throws on unequal-length buffers)

  <action>
    1. `lib/shopify/hmac.ts` — export `verifyShopifyHmac(rawBody: string, hmacHeader: string): boolean` using Node `crypto` (RESEARCH Pattern 4):
       ```ts
       const digest = crypto.createHmac("sha256", process.env.SHOPIFY_API_SECRET!).update(rawBody, "utf8").digest("base64");
       const ok = hmacHeader.length === digest.length &&
         crypto.timingSafeEqual(Buffer.from(hmacHeader), Buffer.from(digest));
       return ok;
       ```
       The `length ===` guard is REQUIRED before `timingSafeEqual` (it throws on unequal-length buffers — a length mismatch must return false, not crash). Read the secret lazily inside the function.
    2. `lib/shopify/hmac.test.ts` — in beforeAll set `process.env.SHOPIFY_API_SECRET = "test_secret_value"`. Compute the correct digest in the test with the same algorithm, assert the four behaviors. This test is the SHOP-05 acceptance since live webhook delivery is deferred (D-11).
